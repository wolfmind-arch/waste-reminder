# API Discovery with Browser DevTools and Postman

The Cyclus API endpoints were identified by inspecting network traffic generated by the official website.

This approach allows the API to be analyzed without relying on existing application code.

---

## Tools Used

* Google Chrome / Brave Developer Tools
* Network tab
* Fetch/XHR filter
* Postman

---

## Step 1 – Open Developer Tools

Open the Cyclus website:

```text
https://cyclusnv.nl
```

Open Developer Tools:

```text
F12
```

Navigate to:

```text
Network
```

Enable the filter:

```text
Fetch/XHR
```

This displays only API requests and hides images, CSS, JavaScript and other resources.

---

## Step 2 – Generate Requests

Enter:

* postcode
* house number

Click:

```text
Meer informatie
```

New HTTP requests appear in the Network panel.

---

## Step 3 – Inspect Requests

For each request inspect:

* Request URL
* HTTP Method
* Query Parameters
* Response
* Headers

This allows the endpoint, request format and JSON structure to be understood.

---

## Step 4 – Import Request into Postman

Right-click the desired request.

Select:

```text
Copy
    └── Copy as cURL
```

Open Postman.

Choose:

```text
Import
```

Paste the copied cURL command.

Postman automatically creates the request with:

* URL
* HTTP method
* Headers
* Parameters

The request can now be executed independently of the browser.

---

## Step 5 – Verify API Response

Execute the request inside Postman.

Verify:

### Endpoint 1

Returns:

```text
Address information
```

Important field:

```text
bagid
```

---

### Endpoint 2

Returns:

```text
Waste collection schedule
```

Important fields:

```text
title
ophaaldatum
```

---

## Benefits of Using Postman

* Test endpoints without using a web browser.
* Verify HTTP responses.
* Inspect JSON responses.
* Save requests in collections.
* Easily test different addresses.
* Reproduce requests during development.
* Simplify debugging before writing application code.

---

## Development Workflow

```text
Browser
     │
     ▼
Developer Tools
     │
     ▼
Network → Fetch/XHR
     │
     ▼
Identify API Request
     │
     ▼
Copy as cURL
     │
     ▼
Import into Postman
     │
     ▼
Verify Request
     │
     ▼
Implement in Python
```

---

## Lessons Learned

* Browser Developer Tools are often the fastest way to discover undocumented APIs.
* Postman is an excellent tool for validating endpoints before writing code.
* Understanding the API first significantly simplifies software development.
* Reverse engineering network requests is a valuable skill for backend and Python developers.
